## 二、本地开发部署

### 2.1 克隆项目

```bash
git clone https://github.com/你的用户名/mimi-cat-showcase.git
cd mimi-cat-showcase
```

### 2.2 安装依赖

```bash
npm install
```

> 如果 `better-sqlite3` 编译失败，安装系统编译工具后重试：
> ```bash
> # macOS
> xcode-select --install
> # Ubuntu/Debian
> sudo apt-get install -y build-essential python3
> # Windows：确保安装 Visual Studio Build Tools 或 Python
> ```

### 2.3 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env`：

```bash
# 后端
APP_ID=你的应用ID
APP_SECRET=至少32位的随机字符串

# 数据库
DATABASE_PATH=./data/app.db

# 前端（Vite 会注入浏览器）
VITE_KIMI_AUTH_URL=https://auth.kimi.com
VITE_APP_ID=你的应用ID

# 后端认证
KIMI_AUTH_URL=https://auth.kimi.com
KIMI_OPEN_URL=https://open.kimi.com

# 管理员（你的 Kimi UnionID）
OWNER_UNION_ID=你的UnionID

# 注册邀请码（可选，留空则无需）
INVITE_CODE=你的邀请码
```

**注意**：`APP_SECRET` 在生产环境必须设置为一个强随机字符串（至少32位），用于 JWT 签名。泄露后攻击者可伪造 Token。

### 2.4 初始化数据库

```bash
npm run db:generate   # 生成迁移文件
npm run db:migrate    # 执行迁移
```

### 2.5 开发模式启动

```bash
npm run dev
```

访问 `http://localhost:5173`

## 三、生产环境部署（VPS）

### 3.1 服务器准备

以 **Ubuntu 22.04** 为例：

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 PM2
sudo npm install -g pm2

# 安装 Nginx
sudo apt install -y nginx

# 安装 Git
sudo apt install -y git

# 安装编译工具（better-sqlite3 需要）
sudo apt install -y build-essential python3

# 创建应用目录
sudo mkdir -p /var/www/mimi
sudo chown -R $USER:$USER /var/www/mimi

# 创建日志目录
sudo mkdir -p /var/log/mimi
sudo chown -R $USER:$USER /var/log/mimi
```

### 3.2 部署代码

```bash
cd /var/www/mimi

# 克隆代码
git clone https://github.com/你的用户名/mimi-cat-showcase.git .

# 安装依赖
npm install

# 配置环境变量
vim .env
# 填入生产环境配置（APP_SECRET 务必设置强密码！）

# 构建前端 + 后端
npm run build

# 首次部署：初始化数据库
npm run db:migrate
```

### 3.3 使用 PM2 启动

```bash
# 使用项目自带的 ecosystem.config.js 启动
pm2 start ecosystem.config.js

# 保存 PM2 配置
pm2 save

# 设置开机自启
pm2 startup systemd
# 执行上面命令输出的指令（类似下面）
sudo env PATH=$PATH:/usr/bin pm2 startup systemd -u $USER --hp $HOME
```

常用命令：

```bash
pm2 status                    # 查看状态
pm2 logs mimi-cat-showcase    # 查看日志
pm2 restart mimi-cat-showcase # 重启
pm2 stop mimi-cat-showcase    # 停止
pm2 monit                     # 监控面板
```

### 3.4 配置 Nginx 反向代理

复制项目自带的 nginx 配置：

```bash
sudo cp deploy/nginx.conf /etc/nginx/sites-available/mimi
```

编辑配置，将 `server_name` 改为你的域名：

```bash
sudo vim /etc/nginx/sites-available/mimi
```

启用配置：

```bash
sudo ln -sf /etc/nginx/sites-available/mimi /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 3.5 配置 SSL（HTTPS）

使用 Let's Encrypt 免费证书：

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 申请证书（替换为你的域名）
sudo certbot --nginx -d mimi.yourdomain.com

# 测试自动续期
sudo certbot renew --dry-run
```

Certbot 会自动修改 Nginx 配置添加 SSL，无需手动配置。

## 四、Docker 部署

### 4.1 快速启动

```bash
# 确保在项目根目录

# 创建数据目录和上传目录
mkdir -p data public/uploads

# 配置环境变量
cp .env.example .env
# 编辑 .env 填入配置

# 构建并启动
sudo docker-compose up -d

# 查看日志
sudo docker-compose logs -f app

# 停止
sudo docker-compose down
```

### 4.2 数据持久化

通过 Docker 卷挂载持久化：

- `./data:/app/data` —— SQLite 数据库文件
- `./public/uploads:/app/public/uploads` —— 上传的图片
- `./dist/public:/var/www/mimi/dist/public:ro` —— 前端构建产物
- `./public/images:/var/www/mimi/public/images:ro` —— 静态图片

### 4.3 更新部署

```bash
# 拉取最新代码
git pull origin main

# 重新构建
sudo docker-compose up -d --build
```

## 五、Nginx 反向代理 + SSL

### 5.1 完整 Nginx 配置（已包含在 deploy/nginx.conf）

```nginx
server {
    listen 80;
    server_name mimi.yourdomain.com;
    return 301 https://$server_name$request_uri;  # HTTP 强制跳转 HTTPS
}

server {
    listen 443 ssl http2;
    server_name mimi.yourdomain.com;

    # SSL 证书（Certbot 自动配置）
    ssl_certificate /etc/letsencrypt/live/mimi.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/mimi.yourdomain.com/privkey.pem;

    # SSL 安全配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers off;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 1d;

    # 安全响应头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/javascript application/xml+rss application/json;

    # 前端静态文件（带缓存）
    location / {
        root /var/www/mimi/dist/public;
        try_files $uri $uri/ /index.html;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # JS/CSS 强缓存
    location ~* \.(js|css|woff|woff2|ttf|otf|eot)$ {
        root /var/www/mimi/dist/public;
        expires 1y;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    # 图片缓存
    location ~* \.(jpg|jpeg|png|gif|webp|svg|ico)$ {
        root /var/www/mimi/dist/public;
        expires 60d;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    # API 代理到 Node 后端
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
        proxy_connect_timeout 60s;
    }

    # 上传文件
    location /uploads/ {
        alias /var/www/mimi/public/uploads/;
        expires 90d;
        add_header Cache-Control "public";
    }

    # 图片目录
    location /images/ {
        alias /var/www/mimi/public/images/;
        expires 90d;
        add_header Cache-Control "public";
    }
}
```

### 5.2 测试并重载

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 六、PM2 进程管理

项目已包含 `ecosystem.config.js`，配置如下：

```javascript
module.exports = {
  apps: [{
    name: "mimi-cat-showcase",
    script: "./dist/boot.js",
    cwd: "/var/www/mimi",
    instances: 1,
    exec_mode: "fork",
    env: { NODE_ENV: "production", PORT: 3000 },
    log_file: "/var/log/mimi/combined.log",
    max_memory_restart: "512M",
    autorestart: true,
    restart_delay: 3000,
    max_restarts: 5,
  }]
};
```

启动后访问：

```bash
pm2 status                    # 查看状态
pm2 logs mimi-cat-showcase    # 查看日志
pm2 restart mimi-cat-showcase # 重启
pm2 stop mimi-cat-showcase    # 停止
pm2 delete mimi-cat-showcase  # 删除
pm2 monit                     # 监控面板
```

## 七、数据库备份

### 7.1 手动备份

```bash
# 直接复制数据库文件（SQLite 支持热备份）
cp /var/www/mimi/data/app.db /backup/mimi-$(date +%Y%m%d).db

# 备份上传的图片
tar -czf /backup/uploads-$(date +%Y%m%d).tar.gz -C /var/www/mimi public/uploads
```

### 7.2 自动备份脚本

创建 `/var/www/mimi/backup.sh`：

```bash
#!/bin/bash
BACKUP_DIR="/backup/mimi"
DB_PATH="/var/www/mimi/data/app.db"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 备份数据库
cp $DB_PATH "$BACKUP_DIR/app_$DATE.db"

# 备份上传的图片
tar -czf "$BACKUP_DIR/uploads_$DATE.tar.gz" -C /var/www/mimi public/uploads

# 保留最近 30 天
find $BACKUP_DIR -name "*.db" -mtime +30 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete

echo "Backup completed: $DATE"
```

添加定时任务：

```bash
chmod +x /var/www/mimi/backup.sh
sudo crontab -e

# 每天凌晨 3 点备份
0 3 * * * /var/www/mimi/backup.sh >> /var/log/mimi-backup.log 2>&1
```

## 八、GitHub Actions 自动部署

### 8.1 创建部署密钥

在服务器生成 SSH 密钥：

```bash
ssh-keygen -t ed25519 -C "github-actions" -f ~/.ssh/github_actions
# 公钥添加到 authorized_keys
cat ~/.ssh/github_actions.pub >> ~/.ssh/authorized_keys
```

### 8.2 配置 GitHub Secrets

在 GitHub 仓库 Settings -> Secrets -> Actions 添加：

| Secret | 值 |
|--------|-----|
| `SSH_HOST` | 服务器 IP 地址 |
| `SSH_USER` | 登录用户名（如 ubuntu） |
| `SSH_KEY` | `~/.ssh/github_actions` 的私钥内容 |

### 8.3 创建工作流

文件 `.github/workflows/deploy.yml`：

```yaml
name: Deploy to Production

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          
      - name: Install dependencies
        run: npm ci
        
      - name: Build
        run: npm run build
        
      - name: Deploy to server
        uses: appleboy/ssh-action@v1.0.0
        with:
          host: ${{ secrets.SSH_HOST }}
          username: ${{ secrets.SSH_USER }}
          key: ${{ secrets.SSH_KEY }}
          script: |
            cd /var/www/mimi
            git pull origin main
            npm install
            npm run build
            pm2 restart mimi-cat-showcase
```

提交后，每次 push 到 main 分支会自动部署。

## 九、常见问题排查

### Q1: better-sqlite3 安装失败

```bash
# Ubuntu/Debian
sudo apt-get install -y build-essential python3

# macOS
xcode-select --install

# 重新安装
npm rebuild better-sqlite3
```

### Q2: 数据库迁移失败

```bash
# 查看状态
npx drizzle-kit status

# 手动推送（开发环境，会修改数据库）
npm run db:push

# 重置（会丢失数据，慎用）
npx drizzle-kit drop
npm run db:migrate
```

### Q3: PM2 启动失败

```bash
# 查看错误日志
pm2 logs mimi-cat-showcase --lines 50

# 检查端口占用
sudo lsof -i :3000

# 确保 .env 存在
ls -la /var/www/mimi/.env
```

### Q4: 图片上传失败

```bash
# 检查目录权限
ls -la /var/www/mimi/public/uploads/

# 修复权限
sudo chown -R $USER:$USER /var/www/mimi/public/uploads
sudo chmod -R 755 /var/www/mimi/public/uploads
```

### Q5: 域名访问 502 Bad Gateway

```bash
# 检查 Node 服务
pm2 status

# 检查 Nginx 配置
sudo nginx -t

# 检查防火墙
sudo ufw status
sudo ufw allow 80
sudo ufw allow 443
```

### Q6: SSL 证书过期

```bash
# 手动续期
sudo certbot renew

# 检查自动续期
sudo certbot renew --dry-run
```

### Q7: 首次登录没有管理员权限

确保 `.env` 中 `OWNER_UNION_ID` 已设置为你的 Kimi UnionID，然后重新登录。
